# Pseudo-Label Tools

This directory contains the core scripts used in our pseudo-label generation pipeline.

## Files

- `prepare_images_for_pseudo.py`  
  Convert the images used for pseudo-label generation into an empty COCO-style JSON file that contains `images` and `categories`, while keeping `annotations` empty.

- `build_pseudo_annotations.py`  
  Filter detector predictions with confidence thresholding and class-wise NMS, then convert the retained predictions into pseudo annotations and merge them with the original few-shot annotations.

For the complete pseudo-labeling pipeline and fine-tuning procedure, please refer to the main repository README.

## Usage

### 1. Prepare images for pseudo labeling

Use `prepare_images_for_pseudo.py` to convert the target image directory into an empty COCO-style JSON file.

Example:

```bash
python tools/pseudo_label/prepare_images_for_pseudo.py \
    --train_dir /path/to/dataset/train \
    --output_json /path/to/output/train_for_pseudo.json \
    --category_id 1 \
    --category_name car
```

Main arguments:

- `--train_dir`: directory containing the images used for pseudo-label generation
- `--output_json`: output path of the generated COCO-style JSON file
- `--category_id`: category id in the generated JSON
- `--category_name`: category name in the generated JSON

### 2. Build pseudo annotations

Use `build_pseudo_annotations.py` to filter the raw detector predictions and merge them with the original few-shot annotations.

Example:

```bash
python tools/pseudo_label/build_pseudo_annotations.py \
    --gt_json /path/to/few_shot/1_shot.json \
    --pred_json /path/to/predictions/base.json \
    --all_train_json /path/to/output/train_for_pseudo.json \
    --output_json /path/to/output/pseudo_round1.json \
    --score_threshold 0.5 \
    --nms_iou_threshold 0.5
```

Main arguments:

- `--gt_json`: original few-shot annotation JSON
- `--pred_json`: prediction JSON generated by the detector
- `--all_train_json`: image JSON generated by `prepare_images_for_pseudo.py`
- `--output_json`: output path of the merged pseudo-label annotation JSON
- `--score_threshold`: confidence threshold for filtering predictions
- `--nms_iou_threshold`: IoU threshold for class-wise NMS

## Notes

- The category setting should be consistent with the few-shot annotations.
- The generated pseudo annotations are merged with the original few-shot annotations to form the training annotations for the next round.
- This process can be repeated for multiple rounds if needed.
